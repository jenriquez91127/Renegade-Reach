# Renegade Reach — Page Behavior Specification

## Core Rule
**There is no hero section. There are no features. There is no scrollable content on load.**

## What Appears on Load
- BC-styled header (logo, navigation — matching bakersfieldcollege.edu)
- BC-styled footer (matching bakersfieldcollege.edu)
- A single minimal centered prompt between header and footer
- Nothing else

## The Trigger
The moment a student types anything OR clicks to start:
- Ren appears and takes over the page
- The entire experience becomes the conversation
- The interface transitions fully into the chat

## What the Chatbot View Looks Like
- Full-page conversation UI
- Ren's messages appear first (warm-up greeting)
- Student types in a clean input area
- No clutter, no sidebar, no extra UI — just the conversation

## Key Design Principle
The minimal pre-chat state is intentional:
- A distressed student does not need to be sold on anything
- They do not need to scroll through features or read about the initiative
- They need one thing: a place to start talking
- The prompt is an invitation, not a pitch
